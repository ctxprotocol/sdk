### L2Book Info Request Example

Source: https://hyperliquid.gitbook.io/hyperliquid-docs/for-developers/api/websocket/post-requests

Example of sending an L2Book info request via the WebSocket API.

```APIDOC
## POST / L2Book Info Request

### Description
Sends a request to retrieve Level 2 order book data for a specific cryptocurrency.

### Method
POST (via WebSocket)

### Endpoint
WebSocket Endpoint

### Request Body
- **method** (string) - Required - Must be "post".
- **id** (number) - Required - Unique request identifier.
- **request** (object) - Required.
  - **type** (string) - Required - Must be "info".
  - **payload** (object) - Required.
    - **type** (string) - Required - Must be "l2Book".
    - **coin** (string) - Required - The cryptocurrency symbol (e.g., "ETH").
    - **nSigFigs** (number) - Required - The number of significant figures for price precision.
    - **mantissa** (null) - Optional - Currently unused or null.

### Request Example
```json
{
  "method": "post",
  "id": 123,
  "request": {
    "type": "info",
    "payload": {
      "type": "l2Book",
      "coin": "ETH",
      "nSigFigs": 5,
      "mantissa": null
    }
  }
}
```

### Response
#### Success Response (200)
- **channel** (string) - "post".
- **data** (object)
 - **id** (number) - The request ID.
 - **response** (object)
 - **type** (string) - "info".
 - **payload** (object)
 - **type** (string) - "l2Book".
 - **data** (object)
 - **coin** (string) - The cryptocurrency symbol.
 - **time** (number) - Timestamp of the data.
 - **levels** (array) - Array of price levels, each containing size and count.

#### Response Example
```json
{
  "channel": "post",
  "data": {
    "id": <number>,
    "response": {
      "type": "info",
      "payload": {
        "type": "l2Book",
        "data": {
          "coin": "ETH",
          "time": <number>,
          "levels": [
            [{"px":"3007.1","sz":"2.7954","n":1}],
            [{"px":"3040.1","sz":"3.9499","n":1}]
          ]
        }
      }
    }
  }
}
```
```

### Order Signed Action Request Example

Source: https://hyperliquid.gitbook.io/hyperliquid-docs/for-developers/api/websocket/post-requests

Example of sending an order signed action request via the WebSocket API.

```APIDOC
## POST / Order Signed Action Request

### Description
Submits a signed order to the exchange via the WebSocket API.

### Method
POST (via WebSocket)

### Endpoint
WebSocket Endpoint

### Request Body
- **method** (string) - Required - Must be "post".
- **id** (number) - Required - Unique request identifier.
- **request** (object) - Required.
  - **type** (string) - Required - Must be "action".
  - **payload** (object) - Required.
    - **action** (object) - Required - Details of the order action.
      - **type** (string) - Required - Must be "order".
      - **orders** (array) - Required - Array of order objects.
        - **a** (number) - Amount.
        - **b** (boolean) - Buy/Sell indicator.
        - **p** (string) - Price.
        - **s** (string) - Size.
        - **r** (boolean) - Reduce only flag.
        - **t** (object) - Time in force details.
          - **limit** (object) - Limit order details.
            - **tif** (string) - Time in force type (e.g., "Gtc").
      - **grouping** (string) - Required - Order grouping (e.g., "na").
    - **nonce** (number) - Required - Nonce for the transaction.
    - **signature** (object) - Required - The digital signature.
      - **r** (string) - Required - Signature component 'r'.
      - **s** (string) - Required - Signature component 's'.
      - **v** (string) - Required - Signature component 'v'.
    - **vaultAddress** (string) - Required - The address of the vault.

### Request Example
```json
{
  "method": "post",
  "id": 256,
  "request": {
    "type": "action",
    "payload": {
      "action": {
        "type": "order",
        "orders": [{"a": 4, "b": true, "p": "1100", "s": "0.2", "r": false, "t": {"limit": {"tif": "Gtc"}}}],
        "grouping": "na"
      },
      "nonce": 1713825891591,
      "signature": {
        "r": "...",
        "s": "...",
        "v": "..."
      },
      "vaultAddress": "0x12...3"
    }
  }
}
```

### Response
#### Success Response (200)
- **channel** (string) - "post".
- **data** (object)
 - **id** (number) - The request ID.
 - **response** (object)
 - **type** (string) - "action".
 - **payload** (object)
 - **status** (string) - "ok".
 - **response** (object)
 - **type** (string) - "order".
 - **data** (object)
 - **statuses** (array) - Array of order statuses.
 - **resting** (object) - Details if the order is resting.
 - **oid** (number) - Order ID.

#### Response Example
```json
{
  "channel": "post",
  "data": {
    "id": 256,
    "response": {
      "type":"action",
      "payload": {
        "status": "ok",
        "response": {
          "type": "order",
          "data": {
            "statuses": [
              {
                "resting": {
                  "oid": 88383,
                }
              }
            ]
          }
        }
      }
    }
  }
}
```
```

### USDC Permit Payload Signing (TypeScript)

Source: https://hyperliquid.gitbook.io/hyperliquid-docs/for-developers/api/bridge2

Example TypeScript code for signing a PermitPayload for batched USDC deposits. Requires wallet client and signature splitting utilities.

```typescript
const payload: PermitPayload = {
  owner,
  spender,
  value,
  nonce,
  deadline,
};

const isMainnet = true;

const domain = {
  name: isMainnet ? "USD Coin" : "USDC2",
  version: isMainnet ? "2" : "1",
  chainId: isMainnet ? 42161 : 421614,
  verifyingContract: isMainnet ? "0xaf88d065e77c8cC2239327C5EDb3A432268e5831" : "0x1baAbB04529D43a73232B713C0FE471f7c7334d5",
};

const permitTypes = {
  Permit: [
    { name: "owner", type: "address" },
    { name: "spender", type: "address" },
    { name: "value", type: "uint256" },
    { name: "nonce", type: "uint256" },
    { name: "deadline", type: "uint256" },
  ],
};

const dataToSign = {
  domain,
  types: permitTypes,
  primaryType: "Permit",
  message: payload,
} as const;

const data = await walletClient.signTypedData(dataToSign);
const signature = splitSig(data);
```

### Order Priority Grouping Example

Source: https://hyperliquid.gitbook.io/hyperliquid-docs/for-developers/api/priority-fees

Orders can be sent with a grouping parameter `p` which is interpreted as a fraction `p / 100000000.0`. This rate is charged from the undelegated staking balance as a fraction of the filled notional, converted to HYPE using the spot mark price. The maximum priority rate is 8bps, corresponding to `p = 80000`.

```json
{"p": 12345}
```

### POST /info - Spot Deploy Auction

Source: https://hyperliquid.gitbook.io/hyperliquid-docs/for-developers/api/info-endpoint/spot

Retrieves information about the Spot Deploy Auction, including auction parameters and state.

```APIDOC
## POST /info - Spot Deploy Auction

### Description
Retrieves information about the Spot Deploy Auction, including auction parameters and state.

### Method
POST

### Endpoint
https://api.hyperliquid.xyz/info

### Parameters
#### Request Body
- **type** (String) - Required - "spotDeployState"
- **user** (String) - Required - Onchain address in 42-character hexadecimal format.

### Request Example
```json
{
  "type": "spotDeployState",
  "user": "0x0000000000000000000000000000000000000000"
}
```

### Response
#### Success Response (200)
- **states** (Array) - Information about the deployed spot tokens.
 - **token** (Integer) - The token identifier.
 - **spec** (Object) - Specification details of the token.
 - **name** (String) - The name of the token.
 - **szDecimals** (Integer) - Decimal places for size.
 - **weiDecimals** (Integer) - Decimal places for wei.
 - **fullName** (String) - The full name of the token.
 - **spots** (Array) - List of spot identifiers.
 - **maxSupply** (Integer) - The maximum supply of the token.
 - **hyperliquidityGenesisBalance** (String) - Genesis balance for hyperliquidity.
 - **totalGenesisBalanceWei** (String) - Total genesis balance in wei.
 - **userGenesisBalances** (Array) - Genesis balances for specific users.
 - **existingTokenGenesisBalances** (Array) - Existing token genesis balances.
- **gasAuction** (Object) - Details about the gas auction.
 - **startTimeSeconds** (Integer) - The start time of the auction in seconds.
 - **durationSeconds** (Integer) - The duration of the auction in seconds.
 - **startGas** (String) - The starting gas price.
 - **currentGas** (String or null) - The current gas price.
 - **endGas** (String) - The ending gas price.

#### Response Example
```json
{
  "states": [
